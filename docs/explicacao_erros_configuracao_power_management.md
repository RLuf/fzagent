## 🔴 LISTA DE COMANDOS QUE FALHARAM E POR QUÊ

### 1️⃣ `systemctl status sleep.target` ❌

```bash
Command failed: bash -lc sudo systemctl status sleep.target
Error code: 4
Message: Unit sleep.target could not be found.
```

**📝 CAUSA:** O target foi **MASKED** (desabilitado)!

Quando um systemd target está masked, o symlink original no `/etc/systemd/` aponta para `/dev/null`. Portanto, quando você executa `systemctl status`, ele não consegue encontrar o arquivo unit real e retorna erro.

**✅ ESTÁ CORRETO:** Isso significa que a hibernação foi desabilitada!

---

### 2️⃣ `systemctl is-masked sleep.target` ❌

```bash
Command failed: bash -lc systemctl is-masked sleep.target
Error: Unknown command verb "is-masked"
```

**📝 CAUSA:** Comando inválido do systemd.

O systemd não possui subcomando chamado `is-masked`. Os comandos válidos são:

- `systemctl list-units --type=target` (para listar todos os targets)
- Verificar o symlink manualmente para `/etc/systemd/...`:
  - `-r /etc/systemd/sleep.target → /dev/null` significa **MASCARADO** ✅

---

### 3️⃣ `systemctl is-enabled sleep.target` ❌ (falhou silenciosamente)

```bash
Command failed: bash -lc systemctl is-enabled sleep.target
```

**📝 CAUSA:** Target está masked, então não é possível verificar status "enabled" através deste comando.

Mas podemos usar `-q` para ver se existe o symlink broken:

- `systemctl list-unit-files --type=target | grep power` mostra todos os targets de energia
